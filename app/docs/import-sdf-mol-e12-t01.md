# SDF/MOL Import (E12-T01)

## Что реализовано
- Добавлена IPC-команда `import_sdf_mol_v1` (Rust backend + TS command key) для импорта `.sdf`/`.mol`.
- Реализован безопасный текстовый парсер `adapters/io/sdf_mol.rs` без выполнения кода и без `eval`.
- Для каждой записи из MOL/SDF рассчитываются:
  - формула по atom block,
  - молярная масса по таблице атомных масс.
- Импортированные вещества сохраняются в локальную БД как `source_type="imported"`.
- Batch импорт сохраняется атомарно в одной транзакции: при ошибке любой записи весь batch откатывается.
- `phase_default` для импортированных веществ в MVP фиксирован как `solid`.
- Идентификаторы импортированных веществ генерируются детерминированно:
  - `imported-substance-{requestId}-{index}`.
- В UI (Library) добавлена кнопка `Import SDF/MOL`, импорт подключен к mutation state (`importing`) с блокировкой контролов.
- После успеха список веществ обновляется, показывается уведомление с количеством импортированных записей.
- Ошибки импорта/парсинга прокидываются в существующий error/notification поток.

## Допущения и ограничения
- `phase_default` всегда `solid` (MVP-допущение).
- Парсер поддерживает формат MOL V2000/MOL-record в SDF на уровне, достаточном для MVP:
  - читает header + counts line + atom block + проверку `M  END`,
  - использует atom symbols только из поддерживаемой таблицы элементов.
- Неизвестный элемент (или некорректная atom line) завершает импорт с `import` ошибкой и контекстом (`file`, `record`, `line`).
- Импорт с дубликатом `(name, formula)` (в файле или в БД) отклоняется с `IMPORT_DUPLICATE_SUBSTANCE`.

## Manual QA checklist
1. Открыть приложение и перейти в `Library`.
2. Нажать `Import SDF/MOL`, выбрать валидный `.mol` с одной записью.
3. Проверить, что появилось уведомление с количеством и вещество добавилось в список с source `imported`.
4. Повторить для `.sdf` с несколькими записями; убедиться, что импортировано ожидаемое количество и у всех `phase=solid`.
5. Попробовать импорт файла с неизвестным элементом (например `Qx`) и проверить, что ошибка содержит `file` + `record` + `line`.
6. Во время импорта проверить, что Library controls (включая кнопку импорта) заблокированы.
7. Убедиться, что create/update/delete продолжают работать после импорта.
