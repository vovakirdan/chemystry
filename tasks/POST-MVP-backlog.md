# Post-MVP Backlog (Detailed)

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## P01 Реализовать side-by-side A/B сравнение сценариев

**Контекст:** Перенесено из MVP по вашему решению; нужно для продвинутого анализа условий.

**Критерии приемки:**
1. Пользователь может открыть два сценария одновременно в режиме сравнения.
2. Метрики различий отображаются синхронно по времени.
3. Доступен экспорт сравнительного отчета.

**Условия:**
- Не ухудшить производительность базового single-scenario режима.

## P02 Добавить продвинутую кинетику и термодинамику

**Контекст:** Требуется рост научной точности после MVP.

**Критерии приемки:**
1. Добавлен отдельный advanced mode с документированными моделями.
2. Пользователь явно видит разницу между baseline и advanced моделями.
3. Новые формулы и ограничения покрыты тестами.

**Условия:**
- Advanced mode не должен ломать совместимость существующих сценариев.

## P03 Расширить набор форматов импорта/экспорта

**Контекст:** После MVP можно добавить дополнительные форматы данных.

**Критерии приемки:**
1. Добавлены минимум 2 новых формата из приоритетного списка.
2. Round-trip тесты проходят для каждого нового формата.
3. UI показывает поддержку форматов и ограничения.

**Условия:**
- Новые парсеры проходят тот же security pipeline, что и MVP форматы.

## P04 Добавить сценарии совместной работы и синхронизацию

**Контекст:** Коллаборация не входит в MVP, но важна для будущего роста.

**Критерии приемки:**
1. Поддержано создание shareable scenario package или облачной синхронизации.
2. Определена модель конфликтов и их разрешения.
3. Реализованы базовые permission-модели для совместной работы.

**Условия:**
- По умолчанию сохранить local-first подход.

## P05 Добавить instructor/learning режим

**Контекст:** Образовательный сегмент выиграет от guided сценариев.

**Критерии приемки:**
1. Добавлены guided labs/templates с шагами.
2. Добавлены критерии проверки результата для студента.
3. Есть экспорт результата для преподавателя.

**Условия:**
- UX режима не должен мешать профессиональному режиму.

## P06 Добавить plugin API для расширяемости

**Контекст:** Плагинная архитектура позволит расширять модели без core-refactor.

**Критерии приемки:**
1. Определен API контракт плагинов.
2. Реализован безопасный lifecycle плагина.
3. Есть минимум один демонстрационный плагин.

**Условия:**
- Изоляция плагинов обязательна, без прямого небезопасного доступа к core.
