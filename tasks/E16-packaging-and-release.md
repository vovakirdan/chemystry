# Epic 16: Packaging and MVP Release

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## E16-T01 Настроить кроссплатформенные release сборки

**Контекст:** MVP должен поставляться для Linux/Windows/macOS.

**Зависимости:** E01-T05, E15-T04.

**Критерии приемки:**
1. CI собирает релизные артефакты минимум для Linux и Windows.
2. Для macOS зафиксирован воспроизводимый pipeline/инструкция сборки.
3. Артефакты подписаны/помечены версией и готовы к публикации.

**Условия:**
- Сборки не должны зависеть от локальных неповторяемых шагов.

## E16-T02 Провести финальный MVP quality gate

**Контекст:** До релиза нужен единый контроль качества по функционалу и техтребованиям.

**Зависимости:** E16-T01.

**Критерии приемки:**
1. Выполнены все обязательные проверки из `00-project-rules.md`.
2. Проверены ключевые acceptance criteria из PRD и task-эпиков.
3. Сформирован релизный checklist с отметками pass/fail.

**Условия:**
- При fail по критичной проверке релиз блокируется.

## E16-T03 Подготовить release notes, quick-start и выпустить v1.0.0

**Контекст:** Пользователю и команде нужен понятный итог релиза и инструкция старта.

**Зависимости:** E16-T02.

**Критерии приемки:**
1. Подготовлены `CHANGELOG`/release notes с перечислением ключевых фич и ограничений.
2. Обновлен quick-start гайд для пользователей.
3. Выпущен тег/релиз `v1.0.0` с приложенными артефактами.

**Условия:**
- Известные ограничения MVP должны быть явно задокументированы.
