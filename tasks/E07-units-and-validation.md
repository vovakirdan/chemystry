# Epic 07: Units and Validation Framework

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## E07-T01 Реализовать модуль единиц и конверсий

**Статус:** done (2026-03-04).

**Контекст:** Расчеты требуют единой, типобезопасной системы единиц.

**Зависимости:** E04-T03.

**Критерии приемки:**
1. Поддержаны конверсии mass <-> mol <-> volume для поддерживаемых кейсов.
2. Единицы нормализуются к внутреннему стандарту перед вычислениями.
3. Ошибки несовместимых конверсий явно сигнализируются.

**Условия:**
- Нельзя делать неявные конверсии без отображения пользователю.

## E07-T02 Внедрить проверку размерностей и нормализацию ввода

**Статус:** done (2026-03-04).

**Контекст:** Неконсистентный ввод — частая причина неверных результатов.

**Зависимости:** E07-T01.

**Критерии приемки:**
1. Добавлена проверка размерностей для всех расчетных формул MVP.
2. Ввод нормализуется (десятичные разделители, пробелы, экспоненциальная форма).
3. Валидация покрыта unit-тестами на граничные случаи.

**Условия:**
- Валидация выполняется до старта дорогостоящих расчетов.

## E07-T03 Реализовать UX-слой ошибок и предупреждений модели

**Статус:** done (2026-03-04).

**Контекст:** Пользователь должен понимать, где ошибка ввода, а где ограничение модели.

**Зависимости:** E07-T02, E03-T04.

**Критерии приемки:**
1. Ошибки и предупреждения отображаются в едином формате UI.
2. Для ограничений модели доступны explain-подсказки в интерфейсе.
3. Повторяющиеся сообщения дедуплицируются и не создают noise.

**Условия:**
- Содержимое сообщений согласовано с терминологией PRD.
