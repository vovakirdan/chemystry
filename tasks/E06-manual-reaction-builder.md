# Epic 06: Manual Reaction Builder (Primary Flow)

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## E06-T01 Реализовать редактор уравнения реакции

**Контекст:** Основной сценарий MVP — ручная сборка реакции.

**Зависимости:** E05-T01, E03-T02.

**Критерии приемки:**
1. Пользователь может добавлять/удалять реагенты и продукты.
2. Для каждого участника можно задать коэффициент и роль.
3. Редактор поддерживает сохранение черновика без запуска симуляции.

**Условия:**
- Должен работать с веществами из библиотеки и user-defined.

## E06-T02 Добавить ввод количеств и фаз по веществам

**Контекст:** Для расчетов и симуляции нужны нормализованные входные данные.

**Зависимости:** E06-T01.

**Критерии приемки:**
1. Для каждого вещества доступны поля mass/mol/volume (где применимо).
2. Доступен выбор фазы вещества.
3. В UI явно показываются единицы измерения и конвертируемые поля.

**Условия:**
- Невалидные комбинации (например, отрицательные количества) блокируют запуск.

## E06-T03 Реализовать pre-run валидацию сценария

**Контекст:** До старта симуляции необходимо ловить ошибки состава и параметров.

**Зависимости:** E06-T02, E07-T01.

**Критерии приемки:**
1. Проверяются обязательные поля, корректность коэффициентов и диапазоны параметров.
2. Ошибки сгруппированы по секциям UI (Builder/Environment/Calculations).
3. После исправления ошибок запуск становится доступным без перезагрузки экрана.

**Условия:**
- Сообщения ошибок должны быть actionable и без внутреннего техжаргона.

## E06-T04 Реализовать сохранение, загрузку и baseline-снимки сценария

**Контекст:** Для what-if анализа нужны сохранения и быстрый возврат к исходным значениям.

**Зависимости:** E06-T02, E04-T04.

**Критерии приемки:**
1. Сценарий сохраняется в локальную БД с уникальным именем/датой.
2. Пользователь может загрузить сохраненный сценарий и продолжить редактирование.
3. Реализован baseline snapshot и действие revert-to-baseline.

**Условия:**
- Формат сохранения должен быть совместим с будущими миграциями.

## E06-T05 Реализовать жизненный цикл симуляции (start/pause/reset)

**Контекст:** Управление прогоном должно быть единообразным для всех режимов.

**Зависимости:** E06-T03, E10-T01.

**Критерии приемки:**
1. Кнопки start/pause/reset доступны и синхронизированы с реальным состоянием симуляции.
2. При reset состояние сцены и расчетов возвращается к baseline.
3. При pause/resume не происходит потери текущего simulation state.

**Условия:**
- Команды управления должны быть идемпотентными.
