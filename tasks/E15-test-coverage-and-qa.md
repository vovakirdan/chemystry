# Epic 15: Test Coverage and QA

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## E15-T01 Сформировать тестовую матрицу и coverage policy

**Контекст:** Нужен явный план, какие слои и сценарии чем тестируются.

**Зависимости:** E08-T04, E10-T04, E12-T05.

**Критерии приемки:**
1. В `docs/testing-strategy.md` описана матрица unit/integration/e2e.
2. Для критичных модулей указаны минимальные требования к покрытию.
3. Определен smoke-набор для релизного прогона.

**Условия:**
- Метрики покрытия не должны стимулировать тесты низкой ценности.

## E15-T02 Добавить unit/integration тесты Rust-домена

**Контекст:** Расчеты и симуляция — наиболее рискованные по регрессиям модули.

**Зависимости:** E15-T01.

**Критерии приемки:**
1. Unit-тестами покрыты calculation и validation модули.
2. Integration-тесты проверяют ключевые сценарии simulation loop.
3. Тесты запускаются в CI без flaky-поведения.

**Условия:**
- Использовать deterministic режим там, где нужны стабильные проверки.

## E15-T03 Добавить unit/component тесты frontend

**Контекст:** UI-логика Builder/Environment/Results должна быть защищена от регрессий.

**Зависимости:** E15-T01, E03-T04.

**Критерии приемки:**
1. Настроен `Vitest` и добавлены тесты ключевых компонентов.
2. Тестируются пользовательские сценарии (ввод, валидация, переключение профилей).
3. Тесты выполняются в CI и формируют отчет.

**Условия:**
- Не тестировать внутренние детали компонентов, только поведение.

## E15-T04 Добавить E2E smoke/regression набор на Playwright

**Контекст:** Нужна проверка целостных пользовательских потоков.

**Зависимости:** E15-T01, E12-T05.

**Критерии приемки:**
1. Есть минимум 3 e2e сценария: manual build run, import/export, settings profile.
2. E2E тесты запускаются headless в CI.
3. При падении тестов сохраняются артефакты (trace/video/screenshot).

**Условия:**
- Сценарии должны использовать стабильные selectors и не быть flaky.
