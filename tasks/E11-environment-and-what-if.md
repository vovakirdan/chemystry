# Epic 11: Environment Controls and What-If Workflow

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## E11-T01 Реализовать контролы температуры, давления и газовой среды

**Статус:** done (2026-03-05).

**Контекст:** Пользователь должен менять условия среды прямо в рабочем процессе.

**Зависимости:** E03-T03.

**Критерии приемки:**
1. Контролы T/P/medium доступны в правой панели.
2. Ввод проходит валидацию диапазонов и единиц.
3. Состояние контролов синхронизируется с simulation engine.

**Условия:**
- Изменения параметров должны быть явно логируемыми в scenario history.

## E11-T02 Реализовать live-derived metrics при изменении условий

**Контекст:** Без мгновенной обратной связи what-if сценарий теряет ценность.

**Зависимости:** E11-T01, E08-T03.

**Критерии приемки:**
1. Derived metrics обновляются при изменении условий в допустимой задержке.
2. Пользователь видит текущие и baseline значения рядом.
3. Ошибочные/невозможные комбинации условий отображаются как warning/error.

**Условия:**
- Обновления должны работать без полной перезагрузки симуляции, где это возможно.

## E11-T03 Реализовать сценарный workflow save/reset/rewind

**Контекст:** Для итеративных экспериментов нужны быстрые операции возврата.

**Зависимости:** E06-T04, E11-T02.

**Критерии приемки:**
1. Доступны операции сохранить состояние, сбросить к baseline, откатить последний шаг.
2. История изменений условий хранится и отображается пользователю.
3. Откат не приводит к рассинхронизации UI и backend state.

**Условия:**
- Хранить историю изменений в локальном persistence-слое.
