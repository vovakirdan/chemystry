## E01-T01 Проверить prerequisites и зафиксировать baseline окружение

**Статус:** done (2026-03-04).

**Контекст:** Перед генерацией проекта нужно зафиксировать инструменты, чтобы сборка была воспроизводимой на Linux/Windows/macOS.

**Зависимости:** нет.

**Критерии приемки:**
1. В `docs/setup.md` записаны минимальные версии Node, Rust, Cargo, Tauri prerequisites.
2. Выбран единый package manager для проекта и это отражено в документации.
3. Добавлена команда быстрой проверки окружения (`scripts/check-env.*` или эквивалент).

**Условия:**
- Следовать [00-project-rules.md](./00-project-rules.md).
- Использовать официальный Tauri v2 quickstart как источник требований.

## E01-T02 Сгенерировать проект через Tauri v2 Quickstart (React + TypeScript)

**Статус:** done (2026-03-04).

**Контекст:** Стартовая структура должна создаваться генератором Tauri, а не вручную.

**Зависимости:** E01-T01.

**Критерии приемки:**
1. Проект создан через `create-tauri-app` с шаблоном React + TypeScript.
2. В репозитории присутствуют корректные `src-tauri/` и frontend-каталоги, созданные шаблоном.
3. Базовые команды `tauri dev` и `tauri build` запускаются без ручных хакающих правок структуры.

**Условия:**
- Не создавать вручную дублирующую структуру до запуска quickstart.
- Все отклонения от шаблона должны быть зафиксированы отдельной задачей.

## E01-T03 Нормализовать структуру проекта поверх шаблона

**Контекст:** После генерации нужна единая структура директорий для будущих эпиков.

**Зависимости:** E01-T02.

**Критерии приемки:**
1. Созданы и задокументированы рабочие каталоги `docs/`, `scripts/`, `tests/e2e/`, `assets/`.
2. Зафиксирован layout frontend-модулей (`features`, `shared`, `app`) и backend-модулей (`domain`, `adapters`, `infra`).
3. В `docs/architecture.md` описано где хранится каждый тип артефакта.

**Условия:**
- Не ломать структуру, сгенерированную quickstart.
- Изменения структуры должны быть обратимыми и понятными новому участнику.

## E01-T04 Настроить единые команды format/lint/test/build

**Контекст:** Исполнителям нужен единый интерфейс команд для локальной и CI-проверки.

**Зависимости:** E01-T03.

**Критерии приемки:**
1. В package scripts добавлены команды `lint`, `test`, `build`, `format`, `format:check`.
2. Для Rust доступны команды форматирования, clippy и тестов через documented aliases.
3. Команды описаны в `README.md` и выполняются на чистом клоне проекта.

**Условия:**
- Команды должны соответствовать quality gate из [00-project-rules.md](./00-project-rules.md).

## E01-T05 Подключить pre-commit и минимальный CI pipeline

**Контекст:** Нужна автоматическая защита от нарушений правил качества.

**Зависимости:** E01-T04.

**Критерии приемки:**
1. Pre-commit проверяет форматирование и линтинг измененных файлов.
2. CI workflow выполняет минимум: format-check, lint, tests, build.
3. Ошибка любой проверки блокирует прохождение pipeline.

**Условия:**
- Не добавлять нестабильные проверки в обязательный pipeline без флага/allow-failure.
