# Epic 08: Calculation Engine

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## E08-T01 Реализовать стехиометрию и лимитирующий реагент

**Статус:** done (2026-03-04).

**Контекст:** Это базовые расчетные функции MVP для учебного применения.

**Зависимости:** E07-T01, E06-T03.

**Критерии приемки:**
1. Модуль рассчитывает стехиометрические соотношения по сбалансированному уравнению.
2. Корректно определяется лимитирующий реагент.
3. Результаты включают входные assumptions и единицы.

**Условия:**
- Для недостающих данных модуль возвращает валидируемую ошибку, а не частичный silent результат.

## E08-T02 Реализовать выход реакции и процент выхода

**Статус:** done (2026-03-04).

**Контекст:** Пользователь должен сравнивать теоретический и фактический выход.

**Зависимости:** E08-T01.

**Критерии приемки:**
1. Вычисляется теоретический выход продуктов.
2. Поддержан ввод фактического выхода и расчет `% yield`.
3. Пограничные случаи (0, отрицательные значения, неполные данные) обрабатываются корректно.

**Условия:**
- Формулы и используемые коэффициенты должны быть явно трассируемы в коде.

## E08-T03 Реализовать концентрации и газовые расчеты (T/P)

**Статус:** done (2026-03-04).

**Контекст:** По PRD обязательны концентрации и газовые расчеты с учетом условий.

**Зависимости:** E08-T01, E11-T01.

**Критерии приемки:**
1. Поддержан расчет концентрации (mol/L).
2. Поддержаны конверсии и расчеты газа с учетом T и P.
3. Все результаты выводятся в стандартизированном Result DTO.

**Условия:**
- Единицы в формулах должны проходить проверку размерностей (из E07).

## E08-T04 Интегрировать результаты в UI, storage и экспорт

**Статус:** done (2026-03-04).

**Контекст:** Расчеты должны быть полезны не только на экране, но и для сохранения/экспорта.

**Зависимости:** E08-T02, E08-T03, E04-T03.

**Критерии приемки:**
1. Результаты отображаются в правой панели с разбивкой по типам расчета.
2. Расчеты сохраняются в `CalculationResult` с метаданными сценария.
3. Доступен экспорт расчетного summary в поддерживаемый локальный формат.

**Условия:**
- Изменение входных данных должно инвалидировать устаревшие результаты.
