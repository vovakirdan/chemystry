# Epic 13: Settings and Performance Management

Обязательное условие для всех задач: строгое соблюдение [Project Rules](./00-project-rules.md).

## E13-T01 Реализовать экран настроек и профили точности

**Контекст:** Пользователю нужны управляемые профили `Balanced`, `High Precision`, `Custom`.

**Зависимости:** E03-T04.

**Критерии приемки:**
1. Реализован экран/модалка настроек с тремя профилями.
2. По умолчанию активен профиль `Balanced`.
3. Переключение профиля обновляет runtime-параметры без перезапуска приложения.

**Условия:**
- Для рискованных профилей показывать предупреждение о потенциальном падении FPS.

## E13-T02 Добавить пользовательские лимиты FPS и количества частиц

**Контекст:** Требование из PRD — дать пользователю контролировать пределы производительности.

**Зависимости:** E13-T01, E10-T01.

**Критерии приемки:**
1. Доступны настройки target FPS и particle limit.
2. Значения валидируются и сохраняются в локальном persistent storage.
3. Ограничения учитываются simulation loop и renderer.

**Условия:**
- Нельзя допускать ввод значений вне безопасного диапазона.

## E13-T03 Внедрить adaptive quality и performance budgets

**Контекст:** Приложение должно деградировать плавно, а не зависать.

**Зависимости:** E13-T02, E09-T03.

**Критерии приемки:**
1. Внедрена стратегия деградации качества при просадке FPS.
2. Заданы performance budgets для ключевых подсистем (render, simulation, IPC).
3. Бюджеты валидированы на baseline железе 16GB RAM / 8 cores / 4GB VRAM.

**Условия:**
- Любое авто-снижение качества логируется и отображается пользователю.
